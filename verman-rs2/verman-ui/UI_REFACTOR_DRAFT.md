# VerMan RS2 UI 重构草案

## 目标

当前界面已经具备核心能力，但页面层级还不够清晰，主要问题是：

- 首页主视觉没有突出“版本管理”这个核心任务
- 信息摘要区被隐藏，用户进入页面后缺少项目全局状态
- 列表项是大段文本拼接，扫描效率偏低
- 详情、对比、设置大量依赖弹层，操作路径偏碎
- Diff 预览可用，但不够像一个版本工具

本次草案的目标不是换皮，而是重构信息架构，让页面更接近：

- 进入页面就能判断当前工作区状态
- 版本历史成为主操作区
- 文件变更成为辅助信息，而不是与版本历史同级竞争
- 详情与对比更连贯，不频繁跳出全屏弹窗

## 重构原则

- 首页只回答三个问题：当前项目是谁、现在变了什么、接下来最可能要做什么
- 版本历史是第一主视图
- 文本信息尽量结构化，不再依赖长字符串堆叠
- 保留桌面工具的效率感，减少“网页化”的空洞留白
- 保持 Slint 可实现性，避免设计脱离当前技术栈

## 新的信息架构

### 首页

页面分为四层：

1. 顶部项目头部
2. 概览统计区
3. 主工作区
4. 轻量状态反馈区

建议结构：

```text
+----------------------------------------------------------------------------------+
| 工作区名                H:\xxx\project                           [切换] [刷新]   |
| 当前状态：有 12 个变更文件，最近版本 v12，备份：开启                           |
+----------------------------------------------------------------------------------+
| [总文件 324] [待提交 12] [版本数 18] [忽略规则 6] [资源管理器集成 正常]         |
+--------------------------------------+-------------------------------------------+
| 版本历史                             | 当前工作区变更                            |
| v18  描述 / 时间 / 文件数            | A src/main.rs      4.2 KB                 |
| v17  描述 / 时间 / 文件数            | M src/ui.rs        8.1 KB                 |
| v16  描述 / 时间 / 文件数            | D temp/data.json   1.2 KB                 |
| ...                                  | ...                                       |
|                                      |                                           |
| [查看详情] [版本对比] [回滚]         | [创建版本] [查看全部]                     |
+--------------------------------------+-------------------------------------------+
| 状态：版本详情已加载                                                     [清除]  |
+----------------------------------------------------------------------------------+
```

### 详情页

详情页不再优先使用全屏弹窗，改为主界面内右侧展开详情区，或独立二级页布局。

建议结构：

```text
+----------------------------------------------------------------------------------+
| 版本 v18  描述：修复预览逻辑                           [返回] [对比] [回滚]      |
| 创建时间：2026-04-11 21:00:12    文件数：12    新增 2 / 修改 9 / 删除 1         |
+------------------------------+---------------------------------------------------+
| 文件列表                     | 文件预览                                          |
| [M] src/main.rs              | src/main.rs                                      |
| [A] README.md                | Previous                     Current             |
| [D] temp/data.json           | 12 | old line                12 | new line        |
| ...                          | 13 | ...                     13 | ...             |
|                              |                                                   |
|                              | 注：二进制文件可使用系统打开                      |
+------------------------------+---------------------------------------------------+
```

### 对比页

对比页建议从“选择 + 结果列表”升级为“选择 + 统计 + 可钻取结果”。

```text
+----------------------------------------------------------------------------------+
| 版本对比：v16 -> v18                                               [关闭]        |
+----------------------------------------------------------------------------------+
| 基准版本 [v16________]   目标版本 [v18________]   [开始对比]                      |
| [新增 2] [修改 9] [删除 1] [共 12 项]                                            |
+------------------------------+---------------------------------------------------+
| 结果列表                     | 差异预览                                          |
| [M] src/main.rs              | 左右对照 / 高亮行 / 行号                          |
| [A] README.md                |                                                   |
| [D] temp/data.json           |                                                   |
| ...                          |                                                   |
+------------------------------+---------------------------------------------------+
```

## 组件重组建议

当前 `main.rs` 中多个区域直接手写 Rectangle + Text，后续维护成本会越来越高。建议拆成以下基础块：

- `AppHeader`
- `SummaryCard`
- `SectionHeader`
- `PrimaryButton`
- `SecondaryButton`
- `StatusChip`
- `VersionRow`
- `ChangeRow`
- `DiffPane`
- `EmptyState`

建议先抽“视觉稳定、复用率高”的组件，不要一开始就拆得太细。

## 首页布局重排建议

### 1. 顶部栏改为双层

当前顶部栏把标题和操作按钮放在一排，页面身份弱，工具感过强。

建议改成：

- 第一层：工作区名、完整路径、当前状态摘要
- 第二层：主要操作按钮

推荐按钮分组：

- 主操作：`创建版本`
- 次操作：`切换项目`、`刷新`、`对比`、`设置`

这样“创建版本”会成为视觉主按钮，其余为辅助。

### 2. 恢复概览区

当前项目摘要区已被隐藏，应恢复并压缩成卡片式概览：

- 总文件数
- 当前变更文件数
- 版本总数
- 忽略规则数
- Explorer 集成状态

这些信息不应写成长句，而应变成一眼能扫到的短标签。

### 3. 调整主次区域权重

建议首页左右比例从现在接近 1:1，改成：

- 左侧 `版本历史` 60%
- 右侧 `工作区变更` 40%

理由：

- 用户后续的详情、对比、回滚都依赖版本历史
- 文件变更只是当前工作区辅助态，不应抢占主视图

## 列表项设计建议

### 版本行

当前版本列表是一行拼接文本，建议改成两行结构：

第一行：

- 版本号
- 描述

第二行：

- 创建时间
- 文件数
- 状态标签

建议示例：

```text
v18    修复版本详情预览和对比逻辑
2026-04-11 21:00:12    12 files
```

选中态建议：

- 左侧竖向强调条
- 整行背景略微抬亮
- 不只靠细边框变化

### 变更行

建议拆成三段：

- 状态标签：A / M / D
- 文件路径
- 文件大小

状态标签建议颜色：

- Added: 绿色
- Modified: 蓝色或橙色
- Deleted: 红色

## Diff 预览建议

当前文本对比主要依赖红色文字高亮，信息仍然偏弱。建议增加：

- 行号
- 整行背景高亮，而不是只改文字颜色
- 左右列标题固定
- 文件路径和类型固定在预览头部
- 二进制文件显示更明确的替代状态

建议规则：

- 修改行：浅黄或浅蓝背景
- 删除侧：浅红背景
- 新增侧：浅绿背景

如果 Slint 当前不方便做复杂 diff，同步滚动可作为第二阶段优化。

## 弹层策略调整

建议保留以下弹层：

- 创建版本
- 设置
- 回滚确认

建议弱化或移除以下全屏弹层：

- 详情
- 对比

替代方案：

- 详情改为主区域内展开
- 对比改为独立工作面板

原因：

- 这两个场景都属于“持续查看 + 多次切换文件/版本”
- 用弹层会不断中断上下文

## 文案与编码建议

当前源码中存在中文乱码痕迹，这会直接影响最终界面观感和布局调试。建议在 UI 重构前先统一：

- 源文件编码使用 UTF-8
- 中英文文案统一收口到单独方法或常量区
- 避免在布局里混入大量长文本判断

建议后续整理为：

- `tr(...)` 保留
- 长文案统一抽成命名清晰的 helper
- 列表行文案改为结构化字段，不再先拼接再显示

## 样式 Token 建议

建议把当前硬编码值收口为一组基础 token，便于统一调样式。

建议变量：

```text
spacing_xs = 4px
spacing_sm = 8px
spacing_md = 12px
spacing_lg = 16px

panel_bg = #f7f9fc
panel_alt = #eef3f8
panel_border = #c7d0db
text_primary = #233041
text_secondary = #5a6a7d
accent = #4f7db3
accent_soft = #dbe8f6

success = #3f8f5d
warning = #c28a2e
danger = #bf4b4b
```

## 分阶段落地方案

### Phase 1

只改首页框架，不动核心逻辑：

- 顶部栏改双层
- 恢复概览卡片
- 重排首页左右布局
- 状态栏缩窄

### Phase 2

重做列表项结构：

- 版本列表改双行结构
- 变更列表加状态标签
- 空状态统一风格

### Phase 3

重做详情区：

- 从弹层改为侧边详情或二级页
- 文件列表和预览区分栏稳定化
- 增加预览头部元信息

### Phase 4

重做对比区：

- 增加统计卡片
- 结果列表支持筛选
- 点击结果直接预览差异

## 与当前代码的映射

当前主要改造点位于：

- `verman-ui/src/main.rs` 顶部工具栏
- `verman-ui/src/main.rs` 被隐藏的项目摘要区
- `verman-ui/src/main.rs` 首页主内容分栏
- `verman-ui/src/main.rs` 详情弹层
- `verman-ui/src/main.rs` 对比弹层
- `verman-ui/src/main.rs` 按钮和容器基础组件

建议先不要一次性全改。最稳妥的路径是：

1. 先抽基础样式和 section header
2. 再改首页
3. 再动详情和对比

## 建议的下一步

如果进入实现阶段，建议先做一个最小可运行版本：

- 不改业务逻辑
- 不改回调定义
- 只调整 Slint 布局结构和视觉层级

这样可以先验证：

- 首页信息层级是否明显改善
- 版本历史是否成功成为主操作区
- 页面是否仍能在 1040 宽度下稳定显示
