# 轻量级本地文件版本管理系统（GUI版）需求报告
## 一、项目概述
本项目旨在开发一款基于 Python 技术栈的轻量级本地文件版本管理系统，定位为「可视化无服务器版简化 SVN」，采用 **PyQt6**（兼顾界面美观与跨平台兼容性）开发 GUI 界面，支持用户通过可视化操作在任意本地文件夹快速创建版本库，实现文件追踪、版本提交、状态查看、历史查询、差异对比、版本回滚等核心功能。系统无需依赖远程服务器或复杂部署，聚焦「本地使用、轻量高效、可视化易用」，适用于不熟悉命令行的个人开发者、办公人员、小团队，弥补传统版本控制工具的学习门槛和操作复杂度。

## 二、核心目标
1.  **无服务器部署**：无需搭建中央服务器，仅依赖本地文件系统完成版本管理；
2.  **可视化操作**：全程通过 GUI 界面完成所有操作，无命令行输入，降低使用门槛；
3.  **文件夹级初始化**：支持通过文件对话框选择任意本地文件夹，一键创建版本库；
4.  **核心功能完备**：覆盖「版本库创建、文件追踪、状态查看、版本提交、历史查询、差异对比、版本回滚、备份清理、配置修改」全流程；
5.  **轻量高效**：安装包体积小、内存占用低，增量存储节省磁盘空间，耗时操作（如大文件提交）不阻塞界面；
6.  **跨平台兼容**：支持 Windows 10+、macOS 12+、Linux（Ubuntu 20.04+、CentOS 8+）；
7.  **界面友好**：布局简洁直观，操作逻辑清晰，状态标识可视化（图标+颜色），降低学习成本。

## 三、用户画像
1.  **不熟悉命令行的个人开发者**：需要本地管理代码、文档，希望通过图形界面快速完成版本控制；
2.  **办公人员**：需追踪报告、配置文件、表格等文档的修改历史，避免误编辑后无法恢复；
3.  **小团队协作（局域网）**：共享 NAS 挂载目录或本地文件夹，通过可视化操作避免文件覆盖，简化版本管理；
4.  **版本控制新手**：希望通过直观界面理解版本控制核心逻辑（追踪、提交、回滚），无需学习命令行；
5.  **追求高效操作的用户**：偏好图形界面的拖拽、点击操作，无需记忆复杂命令。

## 四、功能需求（GUI 可视化版）
### 4.1 版本库初始化模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 选择目标文件夹 | 支持通过「浏览」按钮打开系统文件对话框，选择任意本地文件夹作为版本库根目录 | 点击菜单栏「版本库」→「初始化」→ 弹出文件对话框 → 选择文件夹 → 确认 |
| 一键初始化 | 选中文件夹后，点击「初始化版本库」按钮，自动创建隐藏版本库目录（`.svmini`）及默认配置文件、忽略文件 | 确认目标文件夹后，点击弹窗中的「确认初始化」按钮 → 显示初始化进度（如“创建配置文件中...”）→ 提示“初始化成功” |
| 初始化校验 | 若目标文件夹已存在 `.svmini` 目录，弹窗提示「该文件夹已为版本库，无需重复初始化」；若文件夹只读，提示「无写入权限，请选择其他文件夹」 | 选择文件夹后，系统自动校验 → 弹出对应提示弹窗 → 点击「确定」返回重新选择 |
| 默认配置生成 | 初始化时自动生成 `config.json`（含系统用户名作为默认提交者）和 `.svminiignore`（默认忽略系统临时文件、缓存目录） | 无需用户操作，初始化成功后自动生成，支持后续在「配置」模块修改 |

### 4.2 文件追踪模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 单个/多个文件添加追踪 | 支持通过「添加文件」按钮选择单个或多个文件，或直接拖拽文件到「文件状态」面板，添加至追踪列表 | 1. 点击工具栏「添加追踪」→ 弹出文件对话框 → 选择1个/多个文件 → 确认；<br>2. 从系统文件管理器拖拽文件到「文件状态」面板的「未追踪文件」列表 → 自动添加追踪 |
| 目录添加追踪 | 支持选择整个目录，自动递归追踪目录下所有符合规则（未被忽略）的文件 | 点击「添加目录」→ 选择目录 → 确认 → 系统自动扫描目录下文件（排除忽略规则）→ 提示“成功添加 XX 个文件到追踪列表” |
| 移除文件追踪 | 支持在「文件状态」面板选中已追踪文件，点击「移除追踪」按钮，停止追踪该文件（不删除本地文件） | 在「已追踪文件」列表选中目标文件 → 点击右键菜单「移除追踪」或工具栏「移除追踪」按钮 → 弹窗确认「是否停止追踪该文件？」→ 确认后文件移至「未追踪文件」列表 |
| 文件状态实时显示 | 「文件状态」面板分为三个标签页：「已追踪文件」「未追踪文件」「变更文件」，每个文件条目显示「文件名+路径+状态图标+状态描述」（状态：未追踪/已追踪/已修改/已删除/新增） | 系统自动扫描当前文件夹 → 面板实时更新文件列表，状态图标用颜色区分（如已修改：黄色感叹号；新增：绿色加号；已删除：红色叉号）→ 支持点击「刷新」按钮手动更新状态 |
| 忽略文件配置 | 支持可视化编辑 `.svminiignore` 规则，支持添加、删除、修改忽略条目，支持通配符（如 `*.log`、`__pycache__/`） | 点击菜单栏「设置」→「忽略规则编辑」→ 弹出编辑窗口（左侧为规则列表，右侧为编辑框）→ 点击「添加规则」→ 输入规则（如 `*.tmp`）→ 点击「保存」→ 规则立即生效 |
| 拖拽批量操作 | 支持在「未追踪文件」和「已追踪文件」列表间拖拽文件，快速切换追踪状态 | 选中「未追踪文件」列表中的文件 → 拖拽至「已追踪文件」列表 → 自动添加追踪；反之则移除追踪 |

### 4.3 版本提交模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 变更文件预览 | 提交前在「变更文件」标签页显示所有待提交的变更（新增/修改/删除的已追踪文件），清晰标注文件状态 | 无需手动触发，系统实时扫描已追踪文件 → 「变更文件」列表自动更新，显示文件路径、状态图标、最后修改时间 |
| 提交信息填写 | 强制要求填写提交说明（描述版本变更内容），支持多行输入，提供默认提示文字（如“请输入本次提交的变更说明...”） | 在提交面板的文本框中输入提交信息 → 若未输入，「提交」按钮置灰，提示“请填写提交信息” |
| 选择提交者 | 下拉框显示当前配置的提交者名称，支持直接输入新提交者（自动同步到配置文件） | 提交面板中，点击「提交者」下拉框 → 选择历史提交者或输入新名称 → 确认 |
| 一键提交 | 确认变更文件和提交信息后，点击「提交版本」按钮，系统自动生成版本快照（增量存储），记录版本元数据 | 点击「提交版本」按钮 → 显示提交进度条（如“处理文件：3/10...”）→ 提交成功后弹窗提示「提交成功！版本号：X，提交时间：YYYY-MM-DD HH:MM:SS」→ 「变更文件」列表清空 |
| 提交校验 | 无变更文件时，「提交」按钮置灰，提示“无待提交的变更”；存在未追踪的变更文件时，弹窗提示「存在未追踪的变更文件，是否添加追踪后再提交？」（提供「是/否/取消」选项） | 点击「提交版本」前，系统自动校验 → 弹出对应提示 → 选择「是」则跳转到添加追踪流程，选择「否」则仅提交已追踪的变更，选择「取消」返回 |
| 增量存储优化 | 仅存储与上一版本差异的文件（通过 SHA-256 哈希校验），相同文件复用存储引用 | 无需用户操作，系统后台自动处理，提交完成后在版本历史中显示「新增文件：X 个，修改文件：Y 个，删除文件：Z 个」 |

### 4.4 版本查询模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 版本历史列表 | 「版本历史」面板以表格形式显示所有历史版本，列项包括：版本号（降序排列）、提交时间、提交者、提交信息、变更文件数 | 点击菜单栏「版本」→「查看历史」或工具栏「版本历史」按钮 → 面板自动加载所有版本 → 支持滚动查看，鼠标悬浮显示完整提交信息 |
| 版本详情查看 | 选中任意版本，右侧显示该版本的详细信息：变更文件列表（路径+状态）、每个文件的哈希值 | 点击版本历史表格中的某一行（选中版本）→ 右侧「版本详情」面板自动加载 → 支持展开/折叠文件列表 |
| 当前版本显示 | 在主界面状态栏实时显示当前所在版本号、最后提交时间（如“当前版本：5 | 最后提交：2025-11-04 15:30:00”） | 无需操作，系统启动后自动读取当前版本信息，实时更新状态栏 |
| 版本筛选 | 支持按「提交者」「时间范围」筛选版本，筛选条件支持下拉选择（提交者）和日期选择器（时间范围） | 「版本历史」面板顶部显示筛选栏 → 1. 选择「提交者」下拉框（自动加载所有历史提交者）；<br>2. 点击「开始时间」「结束时间」选择日期 → 点击「筛选」按钮 → 表格仅显示符合条件的版本；<br>3. 点击「重置」恢复显示所有版本 |

### 4.5 差异对比模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 本地与指定版本对比 | 支持选择任意历史版本，对比当前本地已追踪文件与该版本的差异 | 1. 在版本历史列表选中目标版本 → 右键菜单「对比当前版本」；<br>2. 点击工具栏「差异对比」→ 选择「本地 vs 历史版本」→ 下拉选择目标版本 → 确认 |
| 两个历史版本对比 | 支持选择两个不同的历史版本，对比两者间的文件差异 | 1. 按住 Ctrl 键在版本历史列表选中两个版本 → 右键菜单「对比选中版本」；<br>2. 点击「差异对比」→ 选择「历史版本 vs 历史版本」→ 下拉选择两个版本 → 确认 |
| 文本文件可视化对比 | 文本文件采用「分栏显示」：左侧为旧版本内容，右侧为新版本内容，新增行标绿色，删除行标红色，修改行标黄色，支持滚动同步 | 对比后打开「差异对比窗口」→ 顶部显示「对比对象：版本3 vs 版本5」→ 左侧旧版本、右侧新版本 → 鼠标滚轮同步滚动 → 支持复制差异内容 |
| 二进制文件差异提示 | 二进制文件（图片、压缩包等）不显示内容，提示「二进制文件不同」，并显示两个文件的大小、SHA-256 哈希值 | 对比窗口中显示文件路径 → 下方提示「文件类型：二进制 | 旧版本大小：2.5MB | 新版本大小：3.1MB | 哈希值不同」 |
| 差异文件导出 | 支持将文本文件的差异内容导出为 `.diff` 文件，保存到指定路径 | 「差异对比窗口」点击「导出差异」→ 选择保存路径 → 输入文件名 → 确认 → 提示“导出成功” |

### 4.6 版本回滚模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 选择目标回滚版本 | 支持在版本历史列表中选中任意版本作为回滚目标，显示该版本的核心信息（提交时间、提交者、变更文件）供用户确认 | 「版本历史」列表选中目标版本 → 右键菜单「回滚到该版本」→ 弹出「回滚确认窗口」→ 窗口显示目标版本信息及“将覆盖当前已修改的追踪文件”警告 |
| 回滚前备份选项 | 提供「备份当前未提交变更」勾选框（默认勾选），备份文件自动存储在 `.svmini/backups/` 目录，按时间戳命名 | 「回滚确认窗口」勾选/取消「备份当前变更」→ 系统自动生成备份文件夹 → 回滚成功后提示“备份已保存至：XXX路径” |
| 二次确认机制 | 回滚前显示「将被覆盖的文件列表」（当前已修改的追踪文件），要求用户输入“确认回滚”字样或点击「确认回滚」按钮完成二次确认，防止误操作 | 「回滚确认窗口」查看覆盖文件列表 → 点击「确认回滚」按钮 → 弹出二次确认弹窗 → 点击「确定」开始回滚 |
| 回滚进度显示 | 回滚过程中显示进度条，标注“正在恢复文件：file.txt...”，避免用户误以为程序无响应 | 回滚时打开「回滚进度窗口」→ 进度条显示完成百分比 → 实时更新恢复文件名称 → 完成后提示“回滚成功！当前版本已切换到版本3” |
| 回滚撤销 | 支持撤销最近一次回滚操作，恢复到回滚前的状态（依赖最近一次的回滚备份） | 点击菜单栏「版本」→「撤销上一次回滚」→ 弹窗确认「是否撤销回滚？将恢复到回滚前的状态」→ 确认后系统自动从备份目录恢复 → 提示“撤销成功” |

### 4.7 备份清理模块
| 功能点 | 详细描述 | GUI 操作流程 |
|--------|----------|--------------|
| 备份列表查看 | 显示所有回滚备份文件夹（按时间戳排序），标注备份对应的回滚操作（如“回滚到版本3的备份 | 2025-11-04 16:00:00”）、备份文件数、占用空间 | 点击菜单栏「工具」→「备份管理」→ 打开「备份管理窗口」→ 列表显示所有备份信息 |
| 单个备份删除 | 支持选中单个备份，点击「删除选中备份」按钮，删除该备份文件夹（释放磁盘空间） | 「备份管理窗口」选中目标备份 → 点击「删除选中」→ 弹窗提示「确认删除该备份？删除后无法恢复」→ 确认后删除并刷新列表 |
| 批量备份删除 | 支持勾选多个备份，点击「批量删除」按钮，一次性删除选中备份 | 勾选多个备份前的复选框 → 点击「批量删除」→ 二次确认 → 批量删除 |
| 全部备份清理 | 提供「清理所有备份」按钮，一键删除所有备份文件夹（需多重确认） | 点击「清理所有备份」→ 弹窗提示「警告：将删除所有回滚备份，是否确认？」→ 输入“确认清理”字样 → 再次点击「确认清理」→ 清理完成后提示“所有备份已删除” |
| 版本清理（可选） | 支持删除指定历史版本（需确认该版本不是当前版本，且无后续版本依赖） | 「版本历史」列表选中目标版本 → 右键菜单「删除该版本」→ 弹窗提示「删除版本将无法恢复，是否确认？」→ 确认后删除版本文件及元数据 → 版本号自动重新排序（可选，或保留空缺版本号） |

| 帮助文档查看 | 提供内置帮助文档，包含功能介绍、操作步骤、常见问题排查（如“版本库损坏如何修复”） | 点击菜单栏「帮助」→「使用指南」→ 打开帮助窗口（支持分页浏览）；<br>点击「帮助」→「常见问题」→ 显示FAQ列表 |

## 五、非功能需求
### 5.1 性能要求
1.  初始化操作：响应时间 ≤ 2秒（含文件夹扫描、文件创建）；
2.  提交操作：100个小文件（≤1MB）提交响应时间 ≤ 3秒，大文件（≤1GB）提交时显示进度条，无界面假死；
3.  回滚操作：100个小文件回滚响应时间 ≤ 2秒，大文件回滚时进度条实时更新；
4.  增量存储：相同文件重复提交时，磁盘占用不增加，仅存储哈希引用；
5.  界面响应：所有点击、拖拽操作响应时间 ≤ 0.5秒，耗时操作（如扫描大目录、对比大文件）采用多线程处理，主线程不阻塞；
6.  内存占用：运行时内存占用 ≤ 100MB（含GUI渲染+文件处理）。

### 5.2 兼容性要求
1.  操作系统：支持 Windows 10+（32/64位）、macOS 12+、Linux（Ubuntu 20.04+、CentOS 8+）；
2.  Python 版本：基于 Python 3.8+ 开发，打包后无需用户安装 Python 环境；
3.  界面适配：支持不同屏幕分辨率（最小支持 1024×768），控件大小自动适配，无布局错乱；
4.  文件兼容性：支持文本文件（UTF-8、GBK 等编码）、二进制文件（图片、压缩包、视频等），无文件类型限制；
5.  鼠标/键盘操作：支持键盘快捷键（如 Ctrl+S 保存配置、Ctrl+R 刷新状态），兼容不同鼠标操作（左键选择、右键菜单）。

### 5.3 可靠性要求
1.  版本数据安全：提交/回滚过程中异常中断（断电、强制关闭），重启程序后提示「版本库可能损坏，是否尝试修复」，修复工具自动校验并恢复元数据；
2.  文件完整性：每个文件通过 SHA-256 哈希校验，防止文件篡改或传输错误，对比时发现哈希不一致提示「文件完整性校验失败」；
3.  错误处理：所有操作异常（如文件不存在、权限不足、磁盘满）均弹出清晰提示，说明错误原因和解决方案（如“磁盘空间不足，请清理磁盘后重试”），无程序崩溃；
4.  备份可靠性：回滚备份文件独立存储，与版本库文件分离，避免版本库损坏导致备份丢失。

### 5.4 易用性要求
1.  界面布局：遵循「顶部菜单栏+工具栏+左侧功能面板+右侧详情面板+底部日志面板」的经典布局，用户无需学习即可上手；
2.  状态可视化：文件状态、操作结果通过「图标+颜色」标识（如成功：绿色对勾、警告：黄色感叹号、错误：红色叉号），直观易懂；
3.  操作引导：首次启动程序显示「快速入门引导」弹窗（3步操作：创建版本库→添加追踪→提交版本），支持「下次不再显示」；
4.  容错设计：用户误操作（如删除版本、清理备份）时提供多重确认，避免数据丢失；
5.  提示清晰：所有弹窗提示、日志信息使用通俗语言，无技术术语堆砌（如不说“哈希校验失败”，而说“文件已被修改或损坏，无法正常对比”）。

### 5.5 可扩展性要求
1.  模块化设计：核心功能（存储、差异对比、GUI 交互）拆分独立模块，后续可扩展功能（如远程同步、分支管理）无需修改核心代码；
2.  界面插件预留：支持新增功能标签页（如后续添加「分支管理」标签），不影响现有界面布局；
3.  配置可扩展：`config.json` 采用 JSON 格式，支持新增配置项（如自定义备份路径、差异对比颜色），不破坏原有配置结构。

## 八、数据存储设计
版本库核心目录结构与 CLI 版一致（`.svmini` 为隐藏目录），GUI 操作不改变存储逻辑，仅通过可视化界面读写数据：
```
.svmini/
├── config.json        # 系统配置（提交者、忽略文件路径等）
├── index.json         # 追踪索引（已追踪文件路径、哈希值、修改时间）
├── .svminiignore      # 忽略文件规则（用户可通过 GUI 编辑）
├── versions/          # 版本存储目录（按版本号命名文件夹）
│   ├── 1/             # 版本1
│   │   ├── meta.json  # 版本元数据（提交时间、提交者、变更列表等）
│   │   └── files/     # 版本1的增量文件快照
│   ├── 2/             # 版本2
│   └── ...
└── backups/           # 回滚备份目录（按时间戳命名）
```

关键文件格式与 CLI 版一致（JSON 格式），确保数据兼容性，支持 CLI 版与 GUI 版混用（如通过 GUI 创建版本库后，可通过 CLI 提交版本，反之亦然）。

## 十、风险与应对措施
| 风险点 | 可能影响 | 应对措施 |
|--------|----------|----------|
| PyQt6 跨平台兼容性问题 | 不同操作系统下控件样式、布局错乱，或功能异常（如文件对话框显示异常） | 1. 采用 PyQt6 内置的跨平台控件，避免使用系统特定控件；<br>2. 在 Windows、macOS、Linux 三种系统下分别测试，针对不同系统调整样式表；<br>3. 文件对话框使用 PyQt6 的 QFileDialog，而非系统原生对话框，确保一致性 |
| 耗时操作导致 GUI 阻塞 | 大文件提交、目录扫描时界面无响应，用户误以为程序崩溃 | 1. 所有耗时操作（文件扫描、哈希计算、差异对比）放入 QThread 子线程执行；<br>2. 主线程显示进度条或加载动画，实时更新任务状态；<br>3. 子线程与主线程通过信号槽通信，避免直接操作 GUI 控件 |
| 大文件处理性能差 | 1GB 以上大文件提交/回滚时耗时过长，内存占用过高 | 1. 大文件采用分片哈希校验（分块读取计算 SHA-256），降低内存占用；<br>2. 进度条按文件大小分块更新（如“处理中：30%”），提升用户感知；<br>3. 文档提示“单个文件建议不超过 1GB，超大文件可能影响操作速度” |
| 版本库损坏（异常中断） | 提交/回滚过程中断电、强制关闭，导致元数据不完整或文件丢失 | 1. 提交时先写入临时文件（如 `meta.tmp.json`），成功后再替换正式文件；<br>2. 启动程序时自动校验版本库完整性，发现损坏弹出修复提示；<br>3. 修复工具自动备份损坏文件，尝试恢复最近的完整版本元数据 |
| 拖拽操作兼容性问题 | 不同系统下文件拖拽功能失效，或无法识别拖拽的文件 | 1. 采用 PyQt6 的 QDragEnterEvent、QDropEvent 统一处理拖拽事件；<br>2. 支持拖拽文件和目录，自动识别拖拽对象类型；<br>3. 拖拽失败时提示“拖拽失败，请通过「添加文件」按钮操作” |
| PyInstaller 打包后体积过大 | 打包后的可执行文件体积超过 100MB，影响用户下载和安装 | 1. 打包时使用 `--onefile` 参数压缩文件，剔除不必要的依赖库；<br>2. 采用 UPX 工具进一步压缩可执行文件；<br>3. 分平台打包（Windows/macOS/Linux 分别打包），避免冗余文件 |
