## ✨ 软件简介

### 什么是VerMan？

VerMan是一款专为个人用户设计的本地文件版本管理工具，帮助您轻松跟踪文件变化，管理项目版本，安全回滚到任意历史状态。

### 核心功能

- 🔍 **智能变更检测**：自动识别文件的新增、修改、删除
- 📸 **版本快照**：一键创建项目版本快照
- ⏪ **安全回滚**：快速回滚到任意历史版本
- 📊 **版本对比**：直观查看版本间的差异
- 🚫 **智能忽略**：灵活配置不需要管理的文件类型
- 💾 **本地存储**：所有数据安全存储在本地，无需联网

### 适用场景

- 📝 文档版本管理（论文、报告、说明书）
- 💻 代码项目管理（个人开发、学习项目）
- 🎨 设计文件管理（设计稿、素材文件）
- 📚 配置文件管理（系统配置、软件设置）
- 🔬 实验性开发（需要频繁回退的尝试性工作）

## 🎯 基本操作

### 4.1 项目管理

#### 创建新项目

1. **打开软件** → 点击 **项目 → 新建项目**
2. **选择文件夹**：
   - 浏览到您要管理的项目文件夹
   - 建议选择包含重要文档或代码的文件夹
   - 避免选择系统目录（如C:\Windows）
3. **确认创建**：
   - 软件会在文件夹中创建 `.verman.db` 数据库文件
   - 同时创建 `.vermanignore` 忽略文件
   - 首次扫描可能需要几秒钟

#### 打开现有项目

**方法一：通过菜单**

1. 点击 **项目 → 打开项目**
2. 选择包含 `.verman.db` 文件的文件夹

**方法二：通过最近项目**

1. 点击 **项目 → 最近项目**
2. 从列表中选择最近使用过的项目

#### 关闭项目

1. 点击 **项目 → 关闭项目**
2. 确认关闭（这不会删除任何文件）

### 4.2 文件变更管理

#### 查看文件变更

打开项目后，左侧面板会显示所有文件变更：

**变更类型说明：**

- 🔵 **新增**：新创建的文件
- 🟡 **修改**：内容已变化的文件
- 🔴 **删除**：被删除的文件

**示例场景：**

```
假设您正在写一份报告：
┌─────┬─────────────────────────────┐
│状态 │文件路径                     │ │
├─────┼─────────────────────────────┤ │
│新增 │images/chart1.png            │ │
│修改 │report.docx                 │ │
│修改 │data.xlsx                   │ │
│删除 │draft_v1.txt                │ │
└─────┴─────────────────────────────┘ │
```

#### 刷新变更

如果您怀疑软件没有检测到最新变更：

1. 点击界面右下角的 **[刷新]** 按钮
2. 或者按键盘 `F5` 键
3. 软件会重新扫描所有文件

### 4.3 版本操作

#### 创建版本（提交）

当您完成了一些重要的修改后，建议创建一个版本：

1. **检查变更**：确认左侧面板显示的变更符合预期
2. **点击提交**：点击 **[提交版本]** 按钮或菜单 **版本 → 提交版本**
3. **输入描述**：
   ```
   请输入版本描述：
   ┌─────────────────────────────────────┐
   │ 完成第一章的编写和图表制作          │
   │                                     │
   │                                     │
   │          [确定]   [取消]            │
   └─────────────────────────────────────┘
   ```
4. **确认创建**：点击确定完成版本创建

**版本描述建议：**

- ✅ 好的描述："完成用户登录功能开发"
- ❌ 不好的描述："更新了文件"
- ✅ 好的描述："修复了PDF导出错误"
- ❌ 不好的描述："修改"

#### 回滚版本

当您需要恢复到之前的某个版本时：

1. **选择版本**：在右侧版本历史中点击目标版本
2. **点击回滚**：点击 **[回滚选中版本]** 按钮
3. **确认对话框**：
   ```
   确认回滚
   ┌─────────────────────────────────────┐
   │ 确定要回滚到版本 v1.2 吗？           │
   │                                     │
   │ 这将恢复工作区到该版本的状态。       │
   │                                     │
   │        [是]      [否]               │
   └─────────────────────────────────────┘
   ```
4. **备份选择**（如果启用）：
   ```
   备份确认
   ┌─────────────────────────────────────┐
   │ 是否备份当前状态？                   │
   │                                     │
   │ (默认设置: 是)                      │
   │                                     │
   │        [是]      [否]               │
   └─────────────────────────────────────┘
   ```
5. **等待完成**：软件会自动恢复文件状态

**重要提示：**

- 回滚是可逆的，您可以再次回滚到其他版本
- 建议在重要回滚前先备份当前状态
- 回滚不会删除版本记录，只是恢复文件内容

#### 查看版本详情

**方法一：双击版本**
在版本历史中双击任意版本记录

**方法二：右键菜单**

如果您需要将某个版本的所有文件导出到其他位置：

## 🛠️ 高级功能

### 5.1 忽略文件管理

#### 为什么需要忽略文件？

有些文件不需要版本管理，比如：

- 临时文件和缓存
- 编译生成的文件
- 系统文件
- 大型媒体文件
- 个人敏感信息

#### 配置忽略规则

1. **打开设置**：点击 **设置 → 配置设置**
2. **找到忽略文件配置**：

   ```
   忽略文件模式
   ┌─────────────────────────────────────────────────────────────┐
   │ 每行一个文件模式 (支持 * 和 ? 通配符):                    │
   │ ┌─────────────────────────────────────────────────────┐   │
   │ │*.log                                                │   │
   │ │temp/                                                │   │
   │ │*.tmp                                                │   │
   │ │__pycache__/                                         │   │
   │ │.DS_Store                                            │   │
   │ │Thumbs.db                                            │   │
   │ └─────────────────────────────────────────────────────┘   │
   │ ┌──────────────────┐ ┌──────────────────┐               │
   │ │[添加]           │ │[删除]           │               │
   │ └──────────────────┘ └──────────────────┘               │
   └─────────────────────────────────────────────────────────────┘
   ```
3. **添加规则**：

   - 在下方输入框输入文件模式
   - 点击 **[添加]** 按钮
   - 例如：`*.backup`、`temp_files/`、`*.log`
4. **保存设置**：点击 **[确定]** 保存配置

#### 常用忽略模式示例

**基础配置（推荐）：**

```
# VerMan相关文件
.verman.db
.verman_backup/

# 临时文件
*.tmp
*.temp
*.log
*.bak
*.backup

# 系统文件
.DS_Store
Thumbs.db
desktop.ini

# 缓存目录
__pycache__/
node_modules/
.cache/
```

**开发项目配置：**

```
# Python项目
*.pyc
*.pyo
*.egg-info/
venv/
env/

# Web项目
node_modules/
dist/
build/
*.min.js
*.min.css

# IDE文件
.vscode/
.idea/
*.swp
*.swo
```

**文档项目配置：**

```
# Office临时文件
~$*.docx
~$*.xlsx
*.tmp

# PDF相关
*.aux
*.toc
*.out
*.bbl
*.blg

# 备份文件
*~
*.bak
*.orig
```

#### 通配符语法

- `*`：匹配任意数量的字符（除目录分隔符）
  - `*.txt` 匹配所有.txt文件
  - `report*` 匹配以"report"开头的文件
- `?`：匹配单个字符
  - `file?.txt` 匹配 "file1.txt", "filea.txt" 等
- `[]`：匹配字符集中的任意字符
  - `file[0-9].txt` 匹配 "file0.txt" 到 "file9.txt"
- `/`：目录分隔符
  - `temp/` 匹配整个temp目录及其内容

### 5.2 版本比较

#### 比较两个版本

1. **打开比较**：点击 **版本 → 比较版本**

   ```
   版本对比
   ┌─────────────────────────────────────────────────────────────┐
   │ 版本1: [v1.2     ▼] 版本2: [v1.1     ▼] [比较]           │
   ├─────────────────────────────────────────────────────────────┤
   │ [仅在版本1中] [仅在版本2中] [不同的文件]                   │
   │ ┌─────────────────────────────────────────────────────┐   │
   ```

## ❓ 常见问题

### Q1: 为什么有些文件没有显示在变更列表中？

**可能原因：**

1. **文件被忽略了**：检查设置中的忽略文件配置
2. **文件没有变化**：软件只检测实际内容变化的文件
3. **文件权限问题**：确保有读取文件的权限
4. **文件被锁定**：关闭可能占用文件的其他程序

**解决方案：**

1. 检查 **设置 → 配置设置 → 忽略文件模式**
2. 点击 **[刷新]** 按钮重新扫描
3. 确保文件没有被其他程序打开

### Q2: 软件扫描文件很慢怎么办？

**可能原因：**

- 项目包含大量文件（超过1000个）
- 包含大文件（超过100MB）
- 硬盘性能较差
- 杀毒软件干扰

**解决方案：**

1. **添加忽略规则**：忽略不需要管理的文件类型
2. **排除大文件**：将大型媒体文件添加到忽略列表
3. **检查杀毒软件**：将项目文件夹添加到杀毒软件白名单
4. **使用SSD**：如果可能，将项目放在固态硬盘上

### Q3: 回滚版本后文件还是旧的状态？

**排查步骤：**

1. **检查错误提示**：查看是否有错误信息显示
2. **确认文件权限**：确保有写入文件的权限
3. **关闭相关程序**：关闭可能占用文件的程序
4. **手动刷新**：按F5或点击刷新按钮

### Q4: 数据库文件损坏了怎么办？

**预防措施：**

- 定期备份重要的项目文件夹
- 避免强制关闭软件
- 不要直接编辑 `.verman.db` 文件

**应急处理：**

1. **检查备份**：查看 `.verman_backup/` 目录中是否有备份
2. **重建项目**：作为最后手段，可以删除 `.verman.db` 重新创建项目
3. **联系支持**：如数据非常重要，可寻求专业技术支持

### Q5: 可以在同一台电脑上管理多个项目吗？

**答：** 可以！

- 每个项目都是独立的，有自己的数据库
- 可以同时打开多个不同的项目（在不同时间）
- 项目之间完全隔离，互不影响

### Q6: VerMan会占用多少磁盘空间？

**空间占用：**

- **数据库文件**：通常几MB到几百MB，取决于文件数量和大小
- **备份文件**：根据您的备份设置和文件变更频率
- **日志文件**：通常很小，几KB到几MB

**节省空间的建议：**

- 合理配置忽略文件规则
- 定期清理不需要的版本
- 避免版本管理大型二进制文件

### Q7: 软件安全吗？我的数据会泄露吗？

**安全保证：**

- **完全本地**：所有数据存储在您的电脑上
- **无需联网**：软件不需要网络连接
- **数据加密**：数据库使用标准SQLite加密
- **隐私保护**：不会收集或上传任何用户数据

### Q8: 如何迁移项目到其他电脑？

**迁移步骤：**

1. **复制整个项目文件夹**（包含隐藏的 `.verman.db` 文件）
2. **在新电脑上安装VerMan**
3. **打开项目文件夹**
4. **验证所有功能正常**

**注意：**

- 确保复制隐藏文件（可能需要显示隐藏文件）
- 检查文件权限是否正确
- 大型项目可能需要一些时间来加载
