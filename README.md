# 本地文件版本管理工具 (Version Manager)

一款轻量级的本地文件版本管理工具，支持文件夹绑定、文件变更监控、版本记录（含备注）、版本回滚等功能。

## 功能特性

- **项目管理**: 支持新建、打开、关闭和删除项目
- **文件监控**: 自动检测文件变更（新增、修改、删除）
- **版本管理**: 创建版本快照，支持版本描述
- **版本回滚**: 回滚到任意历史版本，支持备份当前状态
- **版本对比**: 比较两个版本之间的差异
- **版本导出**: 将指定版本导出到其他目录
- **图形界面**: 基于Tkinter的简洁GUI界面
- **配置管理**: 支持最近项目记录和个性化配置

## 技术架构

- **语言**: Python 3.8+
- **GUI框架**: Tkinter
- **数据库**: SQLite（每个项目独立的 `.verman.db` 文件）
- **架构模式**: 三层架构（GUI层、业务逻辑层、数据存储层）

## 安装与运行

### 环境要求

- Python 3.8 或更高版本
- 操作系统：Windows、Linux、macOS

### 运行方式

```bash
# 克隆或下载项目
cd verman

# 运行主程序
python main.py
```

### 依赖管理

项目使用标准Python库，无需额外安装依赖：

- `tkinter` - GUI界面（Python内置）
- `sqlite3` - 数据库（Python内置）
- `os`, `hashlib`, `json` - 标准库模块

## 使用流程

### 1. 创建项目

1. 启动程序
2. 点击 "项目" → "新建项目"
3. 选择工作区目录
4. 系统自动在该目录创建 `.verman.db` 数据库文件

### 2. 提交版本

1. 在工作区中创建或修改文件
2. 程序自动扫描并显示变更文件
3. 点击 "提交版本" 按钮
4. 输入版本描述
5. 确认提交

### 3. 版本回滚

1. 在版本历史列表中选择目标版本
2. 点击 "回滚选中版本" 按钮
3. 确认是否备份当前状态
4. 确认回滚操作

### 4. 其他功能

- **版本对比**: 选择两个版本进行比较，查看文件差异
- **版本导出**: 将指定版本的文件导出到其他目录
- **版本详情**: 双击版本行查看详细信息
- **最近项目**: 通过菜单快速访问最近打开的项目

## 数据模型

每个项目包含一个SQLite数据库（`.verman.db`），包含三个主要表：

### config 表（配置信息）
- `key`: 配置项标识
- `value`: 配置值

### versions 表（版本信息）
- `id`: 版本ID（主键）
- `version_number`: 版本号
- `create_time`: 创建时间
- `description`: 版本描述
- `change_count`: 变更文件数量

### files 表（文件快照）
- `id`: 文件记录ID（主键）
- `version_id`: 关联版本ID（外键）
- `relative_path`: 文件相对路径
- `file_hash`: 文件哈希值（MD5）
- `file_status`: 变更类型（add/modify/delete/unmodified）
- `file_content`: 文件二进制内容

## 核心特性

### 精确的变更检测
- 通过MD5哈希值精确识别文件内容变化
- 支持新增、修改、删除三种变更类型
- 跳过系统文件和临时文件

### 完整的版本存储
- 每个版本存储完整的文件状态
- 包含未变更文件的元数据
- 支持准确的版本间对比

### 高效的存储策略
- 只有变更的文件存储内容
- 未变更文件仅存储哈希值
- 平衡完整性和存储效率

## 注意事项

1. **忽略文件**: 系统默认忽略 `.verman.db`、`__pycache__`、`.git` 等文件和目录
2. **文件大小**: 建议用于中小文件（<100MB），大文件可能导致数据库体积过大
3. **性能限制**: 单次扫描文件数建议≤1000个
4. **数据安全**: 版本数据仅存储在本地，请定期备份重要的 `.verman.db` 文件
5. **网络使用**: 本工具仅支持本地版本管理，不涉及网络功能

## 故障排除

### 常见问题

1. **无法打开项目**
   - 确认目录中存在 `.verman.db` 文件
   - 检查文件权限

2. **文件变更检测不准确**
   - 检查文件是否被其他程序占用
   - 确认文件编码格式

3. **版本回滚失败**
   - 确认目标版本存在
   - 检查磁盘空间是否充足
   - 查看错误信息日志
